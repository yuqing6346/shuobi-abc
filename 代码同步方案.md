# 📦 代码自动同步方案

## 🎯 使用场景
- 在公司/学校修改代码并提交到GitHub
- 家里笔记本开机后自动获取最新代码
- **不会自动重启应用**，更新代码后您可以手动决定是否重启

---

## 📋 首次设置步骤

### 1️⃣ 在当前电脑（公司/学校）初始化Git仓库

打开命令行，在项目目录执行：

```bash
# 初始化Git仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "初始提交：英语听写工具"

# 在GitHub上创建仓库后，关联远程仓库（替换YOUR_USERNAME和YOUR_REPO）
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

### 2️⃣ 在家里笔记本克隆仓库

```bash
# 克隆仓库到本地（替换YOUR_USERNAME和YOUR_REPO）
cd D:\
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git 英语听写工具
cd 英语听写工具

# 安装依赖
pip install -r requirements.txt
```

## 🚀 日常使用流程

### 📤 在公司/学校修改代码后：

```bash
git add .
git commit -m "描述你的修改"
git push
```

### 📥 在家里笔记本更新代码：

**方法1：双击脚本（推荐）**
- 直接双击 `自动更新代码.bat`
- 脚本会自动检查网络、拉取最新代码
- **不会重启应用**，请手动决定是否重新运行

**方法2：手动命令**
```bash
git pull
```

## 🔧 进阶功能

### 📌 设置开机自动更新（可选）

如果希望笔记本开机后自动拉取代码：

1. 按 `Win + R`，输入 `shell:startup`
2. 在打开的文件夹中创建 `自动更新代码.bat` 的快捷方式
3. 右键快捷方式 → 属性 → 起始位置：填写项目路径

这样每次开机会自动更新代码（但不会启动应用）。

### 🔐 配置GitHub身份认证（避免每次输入密码）

**推荐使用SSH密钥：**

```bash
# 生成SSH密钥
ssh-keygen -t ed25519 -C "your_email@example.com"

# 复制公钥内容
type %USERPROFILE%\.ssh\id_ed25519.pub

# 在GitHub添加SSH密钥：
# Settings → SSH and GPG keys → New SSH key
```

然后修改远程仓库地址：
```bash
git remote set-url origin git@github.com:YOUR_USERNAME/YOUR_REPO.git
```

## ⚠️ 重要提示

### 🛡️ 用户数据保护
`.gitignore` 已配置忽略以下文件，确保您的学习进度不会被覆盖：
- `game_save.json` - 游戏进度
- `review_list.json` - 复习列表
- `error_records.json` - 错误记录

### 🔄 处理本地修改
如果家里笔记本有未提交的修改，更新脚本会自动暂存：
```bash
# 恢复被暂存的修改
git stash pop
```

### 📱 多设备同步建议
- **代码文件**：通过Git同步
- **学习数据**：可使用云盘（OneDrive/百度网盘）单独同步

## 🆘 常见问题

**Q: 更新时提示"无法连接到GitHub"？**
- 检查网络连接
- 尝试配置代理或使用VPN

**Q: 提示冲突(conflict)？**
```bash
# 查看冲突文件
git status

# 如果家里的修改不重要，直接用远程覆盖
git reset --hard origin/main
```

**Q: 想在家里也能提交代码？**
- 正常使用 `git add` / `git commit` / `git push`
- 下次在公司使用前先 `git pull` 拉取最新代码
