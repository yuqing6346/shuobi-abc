## 🚀 方案1: Hugging Face Spaces（最推荐）

### ✨ 优势
- ✅ **完全免费**，无需信用卡
- ✅ 专为Gradio设计，零配置
- ✅ 永久运行，不会休眠
- ✅ 从GitHub自动同步
- ✅ 提供 `username-spacename.hf.space` 域名

### 📋 部署步骤

#### 1️⃣ 注册Hugging Face账号
访问 https://huggingface.co/join 注册（可用GitHub登录）

#### 2️⃣ 创建Space
1. 访问 https://huggingface.co/new-space
2. 填写信息：
   - **Space名称**: `english-dictation` （或其他名称）
   - **License**: MIT
   - **SDK**: 选择 **Gradio**
   - **Space hardware**: CPU basic (免费)
   - **Visibility**: Public

3. 点击 **Create Space**

#### 3️⃣ 关联GitHub仓库

**方法A: 使用Git推送（推荐）**

```bash
# 添加Hugging Face远程仓库
git remote add hf https://huggingface.co/spaces/YOUR_USERNAME/english-dictation

# 推送代码（需要Hugging Face Token）
git push hf main
```

**方法B: 手动上传文件**
1. 在Space页面点击 **Files** → **Add file**
2. 上传以下文件：
   - `app.py`
   - `word_bank.py`
   - `game_state.py`
   - `requirements.txt`
   - `README-HF.md` （重命名为 `README.md`）

#### 4️⃣ 等待部署
- 上传完成后，Hugging Face会自动构建和部署
- 首次部署约需5-10分钟（下载依赖）
- 部署完成后会显示运行状态

#### 5️⃣ 访问您的应用
- URL格式：`https://YOUR_USERNAME-english-dictation.hf.space`
- 分享给任何人都可以访问！

### 🔐 获取Hugging Face Token（用于Git推送）

1. 访问 https://huggingface.co/settings/tokens
2. 点击 **New token**
3. 选择 **Write** 权限
4. 复制Token
5. 推送时输入：
   - Username: 您的HF用户名
   - Password: Token（不是密码）

## 🚀 方案2: Render（备选方案）

### ✨ 优势
- ✅ 免费层支持Web服务
- ✅ 从GitHub自动部署
- ⚠️ 15分钟无访问会休眠，首次唤醒慢

### 📋 部署步骤

#### 1️⃣ 访问 https://render.com 注册（可用GitHub登录）

#### 2️⃣ 创建Web Service
1. 点击 **New +** → **Web Service**
2. 选择 **Connect a repository**
3. 授权GitHub并选择 `shuobi-abc` 仓库

#### 3️⃣ 配置服务
- **Name**: `english-dictation`
- **Region**: Singapore (最近)
- **Branch**: `main`
- **Runtime**: Python 3
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `python app.py`
- **Instance Type**: Free

#### 4️⃣ 添加环境变量
- `PORT`: `7860`
- `PYTHON_VERSION`: `3.10.0`

#### 5️⃣ 点击 **Create Web Service**

部署完成后获得URL：`https://english-dictation.onrender.com`

## 🚀 方案3: Replit（快速测试）

### 📋 部署步骤

1. 访问 https://replit.com 注册
2. 导入GitHub仓库：

## 📊 数据持久化说明

### ⚠️ 重要提示
免费部署平台的数据存储是**临时的**，重启后会丢失：
- `game_save.json` - 游戏进度
- `review_list.json` - 复习列表
- `error_records.json` - 错误记录

### 解决方案：
1. **推荐**：使用免费数据库（如Supabase、Firebase）
2. **简单**：定期下载数据备份
3. **本地使用**：在家里笔记本上运行，云端仅做演示

## 🆘 常见问题

### Q: 部署后显示"加载模型失败"？
A: 首次运行需要下载KittenTTS模型（约500MB），需要等待5-10分钟

### Q: Render部署后访问很慢？
A: 免费层会休眠，首次访问需要30秒唤醒

### Q: 数据会丢失吗？
A: 免费平台会定期重启，建议在本地使用或接入数据库

### Q: 可以自定义域名吗？
A: Hugging Face/Render免费层不支持，Railway付费版可以
